GitHub Demo

Ship reliable data products, everywhere.

dc43 brings machine-readable data contracts and governance workflows to pipelines, runtimes and catalogs so teams catch regressions earlier, enforce SLAs automatically and keep stewardship visible.

  • Vendor-agnostic lifecycle — author, review and version contracts.
  • Runtime enforcement — CI → runtime checks with one-line helpers.
  • Steward integration — Connect Collibra, Unity Catalog or custom UIs.
Open repo Watch demo
Problem

Expectations (schema, quality, SLAs) are scattered across notebooks, CI, catalogs and steward UIs. Breaks surface late and ownership is unclear.

Solution

dc43 separates contract concepts (lifecycle, drafting, approvals) from implementations (stores, adapters, runtime helpers) and enforces contracts in pipelines while feeding observations back to governance.

Compound demo (Databricks + Collibra)

Create contract → run pipeline (write_with_governance) → metrics posted to Collibra → steward validates → runtime resumes.

Assets used
  • dc43-demo app — contract + DQ UI
